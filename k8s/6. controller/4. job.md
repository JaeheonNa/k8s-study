##### Job으로 생성된 pod는 프로세스가 끝나면 자원을 사용하지 않는 상태로 변경됨.
##### 자동으로 삭제되는 것은 아님. (자동으로 삭제돼 버리면, Job이 끝난 뒤 로그 확인이 불가함.)
##### template에는 특정 작업 수행 후 종료되는 Pod를 정의
##### completions를 설정하면 해당 수만큼의 Pod를 순차적으로 수행한 후 다 끝나면 종료함.
##### parallelism을 설정하면 해당 수만큼 동시에 Pod를 수행함.
##### activeDeadlineSeconds를 설정하면 해당 시간(초) 도래시 Job을 강제 종료함.
##### restartPolicy: Never는 필수값임.

    apiVersion: batch/v1
    kind: Job
    metadata:
        name: job-1
    spec:
        completions: 6
        parallelism: 2
        activeDeadlineSeconds: 30
        template:
            spec:
                restartPolicy: Never
                containers:
                - name; container
                  image: najh0528/hello_8080


##### 이런 Job을 여러 개를 주기적으로 생성하는 object를 CronJob이라고 함.
##### 보통은 CronJob으로 Job을 생성함.
##### concurrencyPolicy는 Allow가 default이며, Allow 설정 시 schedule에 설정된 주기에 따라 Job 수행.
##### Forbid 설정 시, schedule에 설정된 주기가 도래하더라도, 기존에 수행 중인 Job이 존재하면 skip.
##### Replace 설정 시, schedule에 설정된 주기가 도래하면, 기존에 수행 중인 Job을 종료하고 새로운 Job 수행.

    apiVersion: batch/v1
    kind: CronJob
    metadata: 
        name: cron-job
    spec:
        schedule: "*/1 * * * *"
        concurrencyPolicy: Allow
        jobTemplate:
            spec:
                suspend: false
                restartPolicy: Never
                    containers:
                    - name; container
                      image: najh0528/hello_8080

##### 아래 명령어를 통해 주기가 도래하지 않아도 수동으로 CronJob의 Job을 생성할 수 있음.

    kubectl create job --from=cronjob/<CronJop명> <생성할Job이름>
    ex) kubectl create job --from=cronjob/cron-job cron-job-manual-001

##### 아래 명령어를 통해 주기가 도래하더라도 Job이 생성되지 않도록 할 수 있음.
    
    kubectl patch cronjobs <CronJop명> -p '{ "spec" : { "suspend" : true } }'
    ex) kubectl patch cronjobs cron-job -p '{ "spec" : { "suspend" : true } }'
